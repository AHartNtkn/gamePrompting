# Category B: Systems & Emergence

You are evaluating whether the game's systems interact to produce non-obvious outcomes. Focus on how mechanics combine, cascade, and generate dynamics that transcend individual rules.

## Criteria and Evaluation Instructions

### B1. System Interaction Density (0-5)

Map each mechanic in the game and trace which other mechanics it influences or is influenced by. For each mechanic, check:
- Does it read or modify state that another mechanic also reads or modifies?
- Can changes in one system propagate to others, or do systems operate in isolation?
- Draw the dependency graph — is it a web of connections or a set of disconnected nodes?

Score 0 if mechanics are entirely siloed — each system runs independently with no cross-system effects. Score 2 if a few key systems influence each other but most remain isolated. Score 4 if most mechanics are interconnected, with changes in one system routinely affecting multiple others.

### B2. Emergent Behavior (0-5)

Play through several scenarios and look for outcomes that arise from rule interactions rather than being explicitly scripted. Check:
- Do combinations of individually simple rules produce surprising but logically consistent results?
- Are there outcomes the designer clearly didn't hand-author but that follow inevitably from the rules?
- Can the player discover novel strategies or situations by combining mechanics in unexpected ways?

Score 0 if every outcome is directly scripted — nothing happens that wasn't explicitly coded as a specific case. Score 2 if a few emergent interactions exist but most outcomes are pre-authored. Score 4 if the game regularly produces outcomes that are surprising yet logically derivable from the rules, and these emergent situations are a meaningful part of the experience.

### B3. Dynamic Variety (0-5)

Simulate multiple playthroughs or replay the same scenario with different player choices. Check:
- Do different inputs lead to observably different runtime behaviors and game states?
- Does the game funnel all players toward the same experience regardless of choices?
- Are there meaningfully distinct trajectories, or just cosmetic variations on a fixed path?

Score 0 if the game produces effectively the same dynamics every time regardless of player input. Score 2 if some variation exists but most playthroughs follow similar arcs. Score 4 if different player approaches produce genuinely distinct game states, dynamics, and experiences that feel like different games.

### B4. Feedback Loop Health (0-5)

Identify every feedback loop in the game — both positive (amplifying) and negative (stabilizing). For each one, check:
- Is the loop intentional and serving a design purpose?
- Do positive loops have natural limiters preventing runaway dominance?
- Do negative loops prevent permanent stagnation while keeping the game from spiraling?
- Does winning make winning easier without bound (degenerate positive loop), or does the game self-correct?

Score 0 if feedback loops are absent or degenerate — either unchecked positive loops that create unstoppable snowballs, or crushing negative loops that punish any progress. Score 2 if loops exist and mostly function but some are poorly tuned (e.g., a positive loop that takes too long to check, or a rubber-band mechanic that feels arbitrary). Score 4 if positive and negative loops are deliberately balanced, creating dynamic tension — the player can build momentum but faces escalating challenges, and falling behind creates recovery opportunities without feeling artificial.

### B5. Chain Reaction Potential (0-5)

Look for events that cascade through multiple systems in sequence. Check:
- Can a single event trigger consequences in a second system, which in turn triggers a third?
- Are causal chains limited to two links (action → reaction) or do they propagate further?
- Example: drought reduces food production, food shortage causes unrest, unrest triggers rebellion, rebellion weakens defenses, neighboring faction attacks. Does anything like this occur?

Score 0 if every event is locally resolved — an action produces a single direct consequence and nothing further. Score 2 if two-step chains occur occasionally but longer cascades are absent. Score 4 if multi-step causal chains regularly propagate through the game's systems, creating complex situations from simple triggering events.

### B6. Unpredictability from Determinism (0-5)

Examine the game's systems for surprising outcomes that arise without randomness. Check:
- Do deterministic rule interactions produce outcomes the player didn't anticipate?
- Is surprise generated by complexity of interaction rather than by dice rolls?
- If all random elements were removed, would the game still produce novel situations?

Score 0 if all surprise comes from random number generators — deterministic systems produce entirely predictable results. Score 2 if a few surprising deterministic interactions exist alongside heavy reliance on randomness for variety. Score 4 if the game regularly generates surprises through deterministic system interactions, with players unable to fully predict outcomes despite complete information about the rules.

### B7. World Coherence Under Stress (0-5)

Push the game to extreme states — deplete all resources, maximize all values, create extreme imbalances. Check:
- Does the simulation remain internally consistent at the boundaries?
- Do edge cases produce absurd results (negative populations, infinite wealth, impossible states)?
- When multiple systems are under stress simultaneously, does the game degrade gracefully or collapse into nonsense?

Score 0 if extreme states immediately produce incoherent results — the simulation breaks down into absurd or impossible states. Score 2 if the game handles some extremes gracefully but others produce clearly broken behavior. Score 4 if the simulation remains internally consistent even under extreme conditions, with edge cases producing unusual but logically valid outcomes.

### B8. Depth-to-Complexity Ratio (0-5)

Count the number of distinct rules and mechanics, then count the number of meaningfully different decisions they generate. Check:
- How many interesting decisions does each unit of rule complexity produce?
- Could the same decision space be achieved with fewer rules?
- Are there rules that add complexity without adding corresponding depth?
- Compare: a game with 5 rules that generates rich strategy vs. a game with 50 rules that generates the same depth.

Score 0 if the game has high complexity for minimal depth — many rules, few meaningful decisions. Score 2 if the ratio is moderate — there is depth, but the rule set could be leaner. Score 4 if the game achieves rich strategic depth from an elegant, minimal rule set — every rule contributes disproportionately to the decision space.

### B9. Compositional Mechanics (0-5)

Examine whether the game's mechanics function as composable building blocks that the player can combine. Check:
- Can the player combine two or more mechanics to produce an effect that logically follows from each mechanic's rules but isn't explicitly scripted?
- Do mechanics operate like a grammar (verbs that apply to nouns) rather than a fixed list of prescribed interactions?
- Are there tools that compose freely, or does every interaction require its own special-case code?

Score 0 if every interaction is hardcoded — mechanics only combine in ways explicitly authored by the designer, with no room for player-invented combinations. Score 2 if some limited composition exists (a few mechanics combine logically) but most interactions are prescribed. Score 4 if mechanics function as composable primitives — the player can combine them freely, and the results follow logically from each mechanic's rules, producing interactions the designer never explicitly programmed.

### B10. Systemic Predictability (0-5)

Present the player (or simulate) with a novel situation where two or more systems interact in a combination not previously encountered. Check:
- Can the outcome be correctly predicted by reasoning from how each system works individually?
- Do the rules compose in a way that is logically consistent with their individual behavior?
- Or do novel combinations produce arbitrary results that couldn't be deduced from existing knowledge?

Score 0 if novel system interactions produce arbitrary outcomes — knowing how each system works individually provides no guidance for predicting combined behavior. Score 2 if some interactions are predictable from first principles but others produce unexpected or inconsistent results. Score 4 if the player can reliably predict the outcome of never-before-seen system interactions by reasoning from established rules — the systems compose transparently.

### B11. Structural Emergence (0-5)

Examine whether the game's macro-level patterns — phases, difficulty progression, pacing arcs — arise from its systems or are imposed by special-case logic. Check:
- Does the game use hardcoded turn-count triggers, timers, or threshold checks to change difficulty, unlock content, or impose phase transitions?
- If you removed all such scripted triggers, would the game still produce distinguishable early/mid/late phases through natural system dynamics (resource depletion shifting priorities, capability accumulation opening strategies, environmental pressure changing viable actions)?
- Do the systems naturally reach different equilibria over time, or does the game feel the same at turn 30 as turn 5 unless a script intervenes?

Compare: In Project Zomboid, early game is looting-focused because you have nothing, mid game shifts to base-building because you've accumulated enough to defend, late game shifts because scarcity changes character. No trigger — just systems reaching different equilibria. Contrast with a game that plays identically every turn until `if turn >= 15: unlock_advanced_buildings()`.

Score 0 if the game's macro structure is entirely scripted — phases, difficulty, and progression changes are all triggered by hardcoded conditions rather than arising from system dynamics. Score 2 if some structural emergence exists (e.g., resource depletion naturally changes priorities) but the game also relies on scripted triggers for major transitions. Score 4 if the game's phases, difficulty curve, and progression arc arise entirely from system interactions — removing all explicit phase/difficulty scripting would not flatten the experience because the systems themselves produce natural differentiation.

### B12. Simulation Perceptibility (0-5)

For every system the game simulates, check whether its effects are observable through normal play. Check:
- Can the player detect that the system exists without reading documentation or source code?
- Does the system produce visible consequences — changes in output, available options, or game state that the player can observe?
- Are there systems running "under the hood" that never surface in the player's experience?

Score 0 if multiple simulated systems have no player-visible effects — they compute internally but produce nothing the player can observe or react to. Score 2 if most systems produce visible effects but some remain invisible despite being mechanically active. Score 4 if every simulated system produces observable effects that the player can detect, learn from, and factor into their decisions.

### B13. Narrative Prompt Density (0-5) `[CONDITIONAL: game has enough systemic breadth for narrative emergence]`

Examine whether the simulation produces events that invite player interpretation. Check:
- Are events specific enough to be meaningful (involving particular entities, specific causes, concrete consequences)?
- Are events ambiguous enough to leave room for the player to construct their own narrative?
- Does the game over-specify what happened (leaving nothing to interpret) or under-specify (events so generic they prompt no imagination)?

Score 0 if the simulation produces events that are either completely generic ("something happened") or completely narrated ("you felt terror as the dark lord's curse struck you, reminding you of your tragic past") — neither prompts player imagination. Score 2 if some events invite interpretation but most are either too generic or too prescriptive. Score 4 if the simulation consistently produces events that are concrete and specific but narratively open — the player naturally wants to construct a story around what happened.

### B14. Entity Narrative Specificity (0-5) `[CONDITIONAL: game has individually simulated entities]`

When events involve game entities (characters, units, creatures, factions), check whether the events are specific to those particular entities. Check:
- Do entities have distinguishing properties (history, preferences, relationships, skills) that affect how events involving them play out?
- Would the same event involving a different entity produce a different outcome or feel different?
- Or could any entity be swapped for any other without changing the event at all?

Score 0 if entities are interchangeable — events happen TO generic slots, not to specific characters with specific properties. Score 2 if entities have some distinguishing features but these rarely affect events. Score 4 if entities are deeply differentiated and events are specific to the particular entities involved — their histories, properties, and relationships shape what happens and how.
