# Category N: Aesthetic & Thematic Coherence

You are evaluating whether the game has a consistent identity — whether its theme, tone, imagery, and mechanics all serve a unified vision. Focus on how well the game's aesthetic elements reinforce each other and support the player's imaginative engagement.

## Criteria and Evaluation Instructions

### N1. Fantasy Enablement (0-5)

Play through the game and assess whether it creates conditions for make-believe. Check:
- Does the game establish a world the player can imaginatively inhabit, or is it purely abstract?
- Can the player adopt an alternate identity (a character, a ruler, an adventurer), or are they just pushing numbers?
- Do the game's outputs address the player as a participant in the fiction, or as an operator of a system?

Score 0 if the game is entirely abstract with no fictional frame — the player has no identity and nothing to imagine. Score 2 if a fictional frame exists but is thin — the game tells you you're a knight but nothing in the mechanics or output makes you feel like one. Score 4 if the game actively enables make-believe through specific detail, consistent address, and mechanics that map onto fictional actions the player can mentally inhabit.

### N2. Theme Integration (0-5)

Identify the game's stated or apparent theme, then examine each mechanic. Check:
- Does each mechanic reflect or reinforce the theme, or is the theme cosmetic?
- Could you swap the theme for something completely different (e.g., replace "medieval kingdom" with "space colony") without changing any mechanics? If yes, the theme is pasted on.
- Do mechanical decisions feel like thematic decisions? (E.g., in a survival game, does resource scarcity mechanically produce the tension the theme promises?)

Score 0 if the theme is pure decoration — the mechanics have no relationship to it and would work identically with any other theme. Score 2 if some mechanics connect to the theme but others are generic systems that could belong to any game. Score 4 if every mechanic is an expression of the theme — you could not re-theme this game without redesigning its systems.

### N3. Tonal Consistency (0-5)

Read through all of the game's text output, descriptions, and feedback messages. Check:
- Does the writing maintain a consistent emotional register throughout? (E.g., does a grim survival game stay grim, or does it suddenly crack jokes in its error messages?)
- Do mechanical outcomes match the tone? (E.g., a horror game where death is described with the same gravity as buying a potion has tonal whiplash.)
- Do system messages (help text, menus, status displays) maintain the game's voice, or do they break into a neutral technical register?

Score 0 if the tone is incoherent — the game shifts between registers randomly, with grim narration next to flippant system messages next to dry technical output. Score 2 if the main narrative maintains tone but system elements (help, errors, menus) break it. Score 4 if every piece of text the player encounters — narration, feedback, menus, errors — sustains the same tonal identity.

### N4. Unification (0-5)

Step back and assess the game as a whole. Check:
- Does every element (mechanics, text, structure, pacing, theme) feel like it belongs to the same game?
- Or does the game feel like separate pieces glued together — a combat system from one game, a crafting system from another, a narrative style from a third?
- Can you articulate a single unifying vision that explains why every element is present?

Score 0 if the game feels like a patchwork — its parts seem drawn from different design visions with no coherent thread connecting them. Score 2 if most elements cohere but one or two subsystems feel imported from a different game. Score 4 if every element is clearly part of a single vision — nothing feels grafted on, and you could explain why each piece exists in terms of one central idea.

### N5. Holographic Design (0-5)

Examine whether individual elements of the game reflect the whole. Check:
- Does a single mechanic, when examined closely, reveal the game's core themes and priorities?
- Do small details (item descriptions, individual encounter designs, specific rule choices) echo the game's larger design intent?
- Or are individual elements generic — interchangeable with elements from any other game?

Score 0 if individual elements reveal nothing about the game's identity — they are generic building blocks that could appear in any game. Score 2 if some elements reflect the game's character but many are stock components. Score 4 if examining any single element — a mechanic, a piece of text, a structural choice — reveals something about the game's overall design philosophy, as if the whole is encoded in each part.

### N6. Imagination Fuel (0-5)

Read the game's descriptive text — room descriptions, item descriptions, event narration, character introductions. Check:
- Does the text provide specific, concrete, sensory detail that gives the player's imagination material to work with?
- Or is it vague and generic? ("You enter a room. There is an enemy." vs. "You push through a rotting door into a flooded cellar. Something pale moves in the water.")
- Does the level of detail match the importance of the moment? (Major events deserve rich description; routine actions need less.)

Score 0 if descriptions are bare labels with no sensory content — the player is told abstract facts ("a weapon," "a monster," "a place") with nothing to visualize. Score 2 if descriptions exist but are generic or formulaic — they tell you things but don't evoke specific images. Score 4 if descriptions are vivid, specific, and varied — the player can see, hear, and feel the game world because the text gives their imagination concrete material.

### N7. ASCII Art Effectiveness (0-5) [CONDITIONAL: game uses ASCII art]

Examine all ASCII art in the game's output. Check:
- Does the art communicate spatial information (maps, layouts, positions) clearly enough that the player can make decisions based on it?
- Is the art readable — can you tell what it represents without squinting or guessing?
- Does it serve a gameplay function (showing relative positions, terrain, structure) or is it purely decorative?
- Is the character set used consistently (same symbol always means the same thing)?

Score 0 if ASCII art is present but unreadable — the player cannot determine what it represents or extract useful information from it. Score 2 if the art is recognizable but ambiguous in places — some symbols are unclear, or spatial relationships are hard to parse. Score 4 if ASCII art is clear, consistent, and functional — the player can read it at a glance, understand spatial relationships, and use it to inform gameplay decisions.

### N8. Aesthetic Coherence (0-5)

Survey all of the game's visual and textual output as a whole. Check:
- Is there a recognizable style that persists across all outputs? (Consistent formatting, consistent use of whitespace, consistent text density, consistent vocabulary register.)
- Or does the game's presentation vary arbitrarily — dense text blocks in one section, sparse output in another, different formatting conventions in different subsystems?
- If the game uses decorative elements (borders, dividers, ASCII art), are they consistent in style?

Score 0 if the game's outputs have no consistent style — different sections look and read as if produced by different authors with different formatting preferences. Score 2 if a style exists but is inconsistently applied — some sections follow it, others diverge. Score 4 if every output the game produces has a recognizable, consistent aesthetic identity — you could identify a screenshot as belonging to this game from its visual and textual style alone.

### N9. Resonance (0-5)

Assess whether the game creates meaning beyond its mechanical operations. Check:
- Does the game feel like it is about something — a theme, a question, an experience that matters even in a fictional context?
- Do the player's decisions feel like they carry weight beyond their mechanical consequences? (Not just "which option gives more damage" but "what kind of person am I in this world?")
- Does the game leave the player with something to think about, or does it evaporate the moment it's closed?

Score 0 if the game is purely mechanical — it generates no meaning, no reflection, no emotional engagement beyond the immediate puzzle of optimization. Score 2 if moments of meaning emerge occasionally but the game doesn't sustain them — isolated events resonate but the overall experience is hollow. Score 4 if the game consistently creates the feeling that its fiction matters — decisions feel consequential not just mechanically but thematically, and the experience lingers after play ends.

### N10. Functional Space (0-5) [CONDITIONAL: game has spatial design]

Examine the game's spatial design — maps, rooms, areas, terrain. Check:
- Does geography create meaningful gameplay decisions? (E.g., chokepoints that affect combat tactics, resource distribution that motivates exploration, terrain that constrains movement in interesting ways.)
- Or is space purely cosmetic — rooms exist but their layout doesn't matter, distances are irrelevant, geography has no mechanical consequence?
- Does the spatial structure reward exploration and spatial reasoning, or is it just a container for encounters?

Score 0 if space is mechanically irrelevant — rooms or areas exist but their arrangement has no impact on gameplay, and the player could encounter the same content in any order or layout. Score 2 if space has some mechanical effect (e.g., movement costs, adjacency matters) but the specific layout is generic and interchangeable. Score 4 if the spatial design is integral to gameplay — geography creates tactical decisions, exploration is rewarded with meaningful discoveries, and the layout itself is a designed experience that couldn't be randomly shuffled without losing something.
